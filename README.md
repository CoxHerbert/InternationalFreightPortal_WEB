# InternationalFreightPortal_WEB

## 需求概览

### 1. 用户画像清单（目标与痛点）

| 用户类型 | 主要目标 | 典型痛点 |
| --- | --- | --- |
| 货主 | 低成本、透明价格与可控交付；快速拿到订舱与提单；可追踪全程状态 | 询价渠道分散、价格不透明；订舱与资料反复确认；异常沟通慢、状态不可见 |
| 代理 | 高效整合多方资源；降低操作成本；提高客户满意度与复购 | 多平台重复录入；舱位与价格信息滞后；异常处理缺乏统一协同 |
| 承运商 | 提升舱位利用率；降低空舱；稳定的货量与付款 | 需求预测不足；单证不规范导致延误；回款周期长 |
| 内部运营 | 统一流程与合规；降低差错；提升处理效率 | 信息割裂、跨系统协作成本高；异常无法沉淀；人工核对成本高 |

### 2. 端到端流程（系统触点）

| 阶段 | 关键动作 | 系统触点 |
| --- | --- | --- |
| 询价 | 提交起运/目的港、货物信息、时效要求 | 在线询价表单、价格引擎、报价审批、消息通知 |
| 订舱 | 选择报价、锁定舱位、提交订舱资料 | 订舱单、舱位管理、承运商对接、订舱确认通知 |
| 提单 | 提交提单草稿、审核与签发 | 提单草稿、校验规则、电子签发、版本管理 |
| 跟踪 | 运输节点更新、异常预警 | 轨迹追踪看板、节点回传、异常工单 |
| 结算 | 费用确认、对账、开票与支付 | 费用清单、对账单、发票管理、支付/收款状态 |
| 售后 | 异常处理、索赔与复盘 | 客服工单、索赔资料、知识库与复盘报告 |

### 3. MVP 功能清单

**必须有**
- 统一询价入口与报价看板
- 订舱流程（舱位锁定、资料提交、订舱确认）
- 提单草稿与审核流程
- 运输跟踪与异常提醒
- 费用清单与对账导出
- 多角色权限与日志

**可延后**
- 智能推荐路线/承运商
- 预测到港与时效分析
- 可视化 KPI 看板
- 自动化对账与发票对接
- 多语言/多币种

**不做（MVP 阶段）**
- 全链路风控评分
- 深度财务系统集成
- 复杂定制化报表引擎
- 即时竞价撮合
